# Contributing to GECKO

Thanks for your interest in contributing to GECKO!

## Reporting Bugs

- Open a GitHub Issue with a clear title
- Include your OS, Python version, and GECKO version
- Describe what you expected vs what happened
- If possible, include the .kinarm file and trial number or steps to reproduce

## Suggesting Features

- Open a GitHub Issue with the "enhancement" label
- Describe the use case and why it would be useful

## Development Setup

1. Clone the repo: `git clone https://github.com/lukekproj/GECKO.git`
2. Install dependencies: `pip install -r requirements.txt`
3. Run from source: `python gui_main.py`

## Code Style

- Follow existing code conventions in the project
- Use descriptive variable and function names
- Add docstrings to new functions

## Pull Requests

1. Fork the repo and create a branch from `main`
2. Make your changes
3. Test your changes with a .kinarm file before submitting
4. Open a PR with a clear description of what you changed and why

## Questions?

Open an issue or reach out to the maintainers.
